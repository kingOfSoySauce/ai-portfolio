# 6 个月大纲（大模型应用 / RAG / Agent 工程路线）

## Month 1：基础闭环 + 第一个可投项目（RAG v1）

**目标**：从“能跑”到“能解释、能复现、能展示”
**产出**

* RAG 项目 v1（文档上传、切块、向量检索、引用溯源、SSE 流式）
* README 完整（架构图、运行方式、示例、截图/录屏）
* 评估最小集（20–50 条 QA 自动回归）
* 10–20 张面试卡片（RAG 核心问题）

**你要补的知识**

* FastAPI（SSE、异常、依赖注入）
* 向量检索基本套路（top-k、阈值、metadata filter）

---

## Month 2：RAG 工程化 + 检索优化（RAG v2）

**目标**：让回答“更准、更稳、更可控”
**产出**

* 检索优化至少做 2 个：

  * query rewrite / multi-query
  * rerank（或简单重排）
  * hybrid（BM25 + 向量）任选其一
* “检索不到/冲突信息/低置信度”处理策略成体系
* 指标化：引用覆盖率、空引用率、平均耗时、失败率

**你要补的知识**

* chunk 策略（size/overlap/按结构切）
* 评估方法：RAGAS 或 LLM-as-judge（知道怎么用、怎么解释）

---

## Month 3：Agent v1（工具调用 + 多步任务）

**目标**：从“问答系统”升级为“会做事的系统”
**产出**

* Agent 项目 v1（planner/executor 或 ReAct）
* 工具至少 3 个（计算器、搜索/爬取、数据库查询三选二）
* 执行轨迹可视化（每一步：意图→工具→结果→下一步）
* 失败处理：超时、重试、回滚/降级、人工确认（可选）

**你要补的知识**

* Tool schema 设计、参数校验
* 状态管理（session memory / short-term memory）

---

## Month 4：LangGraph / 多 Agent / 可控性（Agent v2）

**目标**：让 Agent “可编排、可调试、可维护”
**产出**

* LangGraph 重写或迁移 Agent 流程（state/node/edge）
* 多 Agent 协作（至少 2 个角色：执行 + 审核/监督）
* 安全与合规：输出约束、敏感内容拦截、引用强制、拒答策略
* 评估：任务完成率、平均步数、失败原因归因

**你要补的知识**

* 上下文工程（prompt 版本管理、上下文裁剪、记忆策略）
* 简单的 guardrails 思路（规则 + 自检）

---

## Month 5：生产化能力补齐（LLMOps 最小集）

**目标**：面试官开始把你当“能上生产的人”
**产出**

* 成本控制：token 统计、缓存策略、模型降级策略
* 可观测：日志结构化、trace、关键指标面板（简单也行）
* 部署：Docker 化 + 一键启动（或云部署 demo）
* 文档完善：API 文档、架构图、故障演练说明

**你要补的知识**

* 限流、队列/异步任务（了解即可）
* 常见线上问题：超时、并发、上下文过长、数据脏

---

## Month 6：求职冲刺与定向补洞（作品集 → Offer）

**目标**：稳定拿面试、能讲清楚系统设计与权衡
**产出**

* 作品集最终版：RAG v2 + Agent v2 +（可选）Prompt/评测小项目
* 面试材料：

  * 系统设计题 2 套（RAG、Agent）
  * 常问点 Q&A（50 题以内，全部可口述）
* 投递节奏与复盘：每周投递/面试/迭代简历与项目

**你要补的知识**

* 结合目标岗位 JD 定向补：Milvus/Redis/消息队列/权限等（按需）

---

# 贯穿 6 个月的“产出节奏”（最重要的执行规则）

每个月必须固定交付：

* **1 次项目版本升级**（v1→v2→可生产）
* **1 次评估/指标更新**（让改动可证明）
* **1 次面试材料扩充**（卡片 + 系统设计）
* **1 次展示物更新**（README/录屏/截图）

---

# 你最终会得到的“可投简历资产包”

* 项目 1：企业知识库 RAG（引用溯源 + 评估 + 工程化）
* 项目 2：任务型 Agent（LangGraph 编排 + 工具调用 + 可视化轨迹）
* 可选项目 3：Prompt/评测实验室（A/B、回归测试、版本管理）
